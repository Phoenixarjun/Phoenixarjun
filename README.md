## What I Build

I specialize at the intersection of **backend systems** and **AI infrastructure** — designing systems that are observable, fault-tolerant, and ready for production from day one.

My current focus areas:

- **Event-driven microservices** — Kafka-based pipelines, SAGA orchestration, CQRS, Outbox + CDC patterns
- **AI systems** — RAG pipelines, multi-agent orchestration, LLM routing, prompt engineering infrastructure
- **ML infrastructure** — Feature stores, ONNX inference, drift detection, model lifecycle pipelines
- **Open source** — Building tools that improve how engineers and AI systems work together

## 📦 Featured Projects

### 🧠 [Architecture as Memory (AAM)](https://github.com/Phoenixarjun/architecture-as-memory)
> **Published open-source package** — npm · PyPI · Cargo

A YAML-based architectural boundary system for AI-native repositories. Solves the context drift problem in AI-assisted development — give your AI assistant a persistent, structured understanding of your codebase architecture so it stops making decisions that violate system boundaries.

`YAML` `Architecture` `AI-native tooling` `Multi-runtime`

### ⚡ [PromptNova](https://prompt-nova.vercel.app/)
> **Open-source prompt engineering platform**

Multi-agent fan-out architecture that runs a prompt through 25+ prompting strategies simultaneously — chain-of-thought, few-shot, role-based, constraint injection, and more. Built to help engineers and teams systematically improve their LLM prompts rather than guessing.

`LangChain` `LangGraph` `Next.js` `Multi-agent` `Prompt engineering`
