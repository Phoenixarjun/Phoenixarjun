### 🧠 Architecture as Memory (AAM)

> **Published open-source package** · [npm](https://www.npmjs.com/package/architecture-as-memory) · [PyPI](https://pypi.org/project/architecture-as-memory/) · [Cargo](https://crates.io/crates/architecture-as-memory) · [Live Demo](https://architectureasmemory.vercel.app)

| | |
|---|---|
| **Situation** | AI coding assistants like Copilot and Claude lack persistent awareness of project architecture — they repeatedly make decisions that cross service boundaries, violate domain contracts, or ignore existing patterns. |
| **Task** | Build a lightweight, framework-agnostic system that gives any AI assistant a structured, always-available understanding of a codebase's architecture. |
| **Action** | Designed a YAML-based architectural boundary specification that works as an in-repo config. Shipped native parsers in three runtimes (Node.js, Python, Rust) so it integrates into any AI-native workflow regardless of stack. |
| **Result** | Published simultaneously to npm, PyPI, and Cargo. Live documentation site deployed on Vercel. First open-source package shipping across three package ecosystems in 2 days. |

`YAML` `Node.js` `Python` `Rust` `AI-native tooling` `Open Source`

### ⚡ PromptNova

> **Open-source prompt engineering platform** · [Live](https://prompt-nova.vercel.app/) · [GitHub](https://github.com/Phoenixarjun/PromptNova)

| | |
|---|---|
| **Situation** | Engineers using LLMs pick prompting strategies arbitrarily — chain-of-thought here, few-shot there — with no systematic way to compare what actually works for a given task. |
| **Task** | Build a platform that removes the guesswork from prompt engineering by running multi-strategy comparison at scale. |
| **Action** | Designed a multi-agent fan-out architecture where a single input prompt is dispatched to 25+ parallel agents, each applying a different prompting strategy (chain-of-thought, few-shot, role-based, constraint injection, Socratic, etc.). Results are ranked by quality metrics. |
| **Result** | Platform live on Vercel. Engineers can now empirically compare prompt strategies instead of relying on intuition. |

`LangChain` `LangGraph` `Next.js` `Multi-agent` `Prompt engineering`
